Implementation on any website

Having Car Battle on your site is just a while. If you don't have an account, register and once you're logged in register new partner at http://www.car-battle.com{% url "partners" %}.


Now all you need to do is to put given HTML code on freshly created page on your website. That's all step necessary.



Implementation on sites that require authentication

Our goal is to provide as easy and painless integration of Car Battle with your website as possible. If your users are already signed in with your website there's no reason why they should be bothered with another forms. But to achieve that we requires simple form of authentication between our websites.


Follow these steps:

  1. Register first,
  2. Go to http://www.car-battle.com{% url "partners" %} and create partner's profile with name of your choice,
  3. Create a new page on your website with width of 760px or more,
  4. Put given HTML (iframe) code on created page
  5. add 3 more parameters to an iframe's src (url) section:
    • username - username of currently signed in user
    • password - MD5 of combined username (above) and secret code we gave you upon creating partner profile
    • email - email of currently signed in user

    complete iframe can look like this:
    <iframe border="0" frameborder="no" framespacing="0" scrolling="no" style="width: 760px; height: 2500px; overflow: hidden;" src="http://partner.car-battle.com/?partner=test&username=axel_f&email=axel_f@gmail.com&password=3n23nrff03das0a2"></iframe>


Obviously if you signing in is just an option for you, using both forms of authentication with one partner is technically possible. Anyway, we recommend to convince or even force users to sign in before they play, as in effect you'll create two or more game accounts. It may confuse some players.
